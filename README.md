# Задание 1



## Задача 1



### Текст задачи



Получить последовательность из произведений цифр соответствующих чисел, хранящихся в исходной последовательности.



### Алгоритм решения



1. Запрашиваем количество элементов последовательности у пользователя.
2. Проверяем, что количество элементов > 0.
3. Создаем последовательность с заданным количеством элементов.
4. С помощью Seq.map (применение функции к каждому элементу, возвращение измененной последовательности) каждый элемент:

    1. Проверяем на частный случай (0), возвращаем 0.

    2. Иначе вычисляем произведение цифр в элементе и возвращаем полученный результат, как элемент новой последовательности.

4\. Выводим новую последовательность.

# Задание 2



## Задача 1



### Текст задачи



Последовательность содержит строки, обозначающие двоичные числа от 1 до 9. Найти в десятичной системе счисления сумму элементов последовательности.



### Алгоритм решения



1. Запрашиваем количество элементов последовательности у пользователя.
2. Проверяем, что введенное количество элементов > 0.
3. При создании последовательности:

    1. Проверяем, что пользователь вводит бинарные значения.

    2. Проверяем, что бинарное значение в диапазоне от 1 до 9.

3\. При вычислении десятеричной суммы бинарных значений:

    1. Преобразуем бинарное значение в десятеричное (число).

    2. С помощью Seq.fold (сведение к одному значению) приводим все полученные числа к одному значению (сумме).

4\. Выводим полученную сумму.

# Задание 3



## Задача 1



### Текст задачи



Выяснить, есть ли в каталоге и его подкаталогах файл с введенным 

названием. 



### Алгоритм решения



1. Запрашиваем путь к каталогу у пользователя.
2. Проверяем, что введенный путь существует.
3. Запрашиваем название файла (полностью) у пользователя.
4. Проверяем, что не введена пустая строка.
5. При поиске файла в каталоге:

    1. Собираем все файлы каталога и подкаталогов в одной последовательности.

    2. С помощью Seq.tryFind ищем заданное название файла в полученной последовательности.

3\. Если файл найден:

    1. Выводим первое вхождение найденного файла.

    2. Иначе выводим сообщение о том, что файл не найден..
